# v1.0.77 curl+bash Installer Source and Smoke Release Note

Sayane tightens the current `curl | bash` primary install path so operators and
maintainers can verify the path more directly while keeping the runtime policy
unchanged.

The primary supported install path remains:

```bash
curl -fsSL https://raw.githubusercontent.com/zyx-corporation/sayane/main/scripts/install.sh | bash
```

Native macOS app install remains a secondary operator path layered on top of
that CLI / Bridge runtime.

## Included

- `scripts/install.sh`
  - supports `SAYANE_INSTALL_SOURCE=auto|pypi|git`
  - defaults to `auto`
  - prefers PyPI for the ordinary one-line install
  - switches to git automatically when `SAYANE_REF` is explicitly pinned
  - prints clearer end-of-install quickstart guidance for the CLI / Bridge path
- `scripts/check-install-local.sh`
  - runs a local smoke check for the primary installer path
  - uses temporary install/bin/home directories
  - installs from the local checkout
  - verifies wrapper creation, profile initialization, and quickstart output

## Current boundary

- ordinary operator install is still local-only
- PyPI-first does not remove git fallback; it just avoids requiring git for the
  default path when PyPI is sufficient
- native macOS app packaging, signing, and notarization remain separate from
  this installer slice

## Verification used in this slice

```bash
bash -n scripts/install.sh scripts/check-install-local.sh
bash scripts/check-install-local.sh
```

## Operator outcome

The shortest install path is now easier to explain and easier to verify:

1. run `curl | bash`
2. confirm `sayane --version`
3. start `sayane serve` when Bridge is needed
4. use `bash scripts/check-install-local.sh` from a repo checkout when
   maintainers want a local smoke check for the same primary path
