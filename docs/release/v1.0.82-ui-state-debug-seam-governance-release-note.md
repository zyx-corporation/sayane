# v1.0.82 UI State Debug Seam Governance

Date: 2026-06-30

## Summary

The macOS-native-first line now records the long-term governance decision for the retained
cookie-backed debug shell JSON routes.

## What changed

- accepted ADR 0028 to keep `/app/ui-state/*` and `/app/ui-action/*` as maintainer-only debug seams
- clarified that operator-facing native app integrations continue to prefer bearer-backed `/app/*`
- updated bootstrap architecture, integration contract, bridge manual, and roadmap notes to match

## Why it matters

This closes the remaining ambiguity around whether the cookie-backed JSON shell routes had to retire
before macOS-native completion could be considered stable. They do not: they remain bounded debug
transport, while routine product UX stays native-first.

## Follow-up

- continue shrinking routine references to server-rendered `/app/ui` HTML pages
- decide later whether those HTML pages can retire entirely without losing maintainer diagnostics
