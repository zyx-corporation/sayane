# v1.0.46 Native Daemon Handoff CLI Diagnostics Release Note

## Summary

The native macOS daemon handoff export now includes launchctl print and stdout/stderr tail commands for first-pass CLI diagnostics.

## What changed

- added `launchctl print` command to the exported handoff note when available
- added `stdout tail` and `stderr tail` commands from the LaunchAgent preview payload
- kept the export local-first, plain-text, and read-first

## Operator impact

- exported handoff notes now point directly to the first CLI commands for daemon diagnosis
- log-follow and launchctl inspection no longer require reopening the app to find the exact commands
- handoff recipients can start with browser, app, or CLI paths from the same artifact

## Validation

- `swift build --package-path macos/SayaneApp`
- focused `swift test --package-path macos/SayaneApp --filter ...` still intermittently stalls after test bundle link through `swiftpm-xctest-helper` in this environment
