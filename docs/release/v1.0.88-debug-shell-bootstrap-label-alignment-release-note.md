# v1.0.88 Debug Shell Bootstrap Label Alignment Release Note

## Summary

Routine-facing operator status output now labels the retained `/app/ui` entry URL as
`debug_shell_bootstrap_ui` rather than a generic `bootstrap_ui`.

## What changed

- `sayane app daemon-operator-phase-status` text output now prints `debug_shell_bootstrap_ui`
- operator phase JSON now includes `debug_shell_bootstrap_ui` while preserving `bootstrap_ui` for
  compatibility
- native model decoding accepts the new field name and falls back to the old one
- active CLI/reference docs now describe `/app/ui` as a maintainer/debug compatibility shell more
  explicitly

## Why

P6/M12 is shrinking routine operator dependence on the retained browser compatibility surface.
Calling the URL a generic “bootstrap UI” made the debug shell look closer to a product-facing
routine path than it is.
