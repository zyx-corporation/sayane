# v1.0.61 Resident App CLI Serve Watchfiles Guard Release Note

## Summary

- guarded `sayane serve` so normal Bridge startup no longer depends on a broken
  `watchfiles` native extension import path
- kept `mcp` package imports lazy so ordinary CLI startup does not pull server-only
  native dependencies
- hardened resident app local launcher and smoke helpers to stop stale
  non-listening `serve` processes before restart

## Why

- the repo-local resident app and smoke path must stay usable even when optional
  native wheels in the development environment are damaged or code-sign-invalid
- operator startup scripts should recover from stale detached Bridge processes
  instead of waiting on a dead port forever

## Validation

- `PYTHONPATH="$PWD/src${PYTHONPATH:+:$PYTHONPATH}" .venv/bin/python -m sayane.cli.main --version`
- `PYTHONPATH="$PWD/src${PYTHONPATH:+:$PYTHONPATH}" .venv/bin/python -m sayane.cli.main serve --help`
- `bash scripts/check-resident-app-release-smoke.sh --start`
