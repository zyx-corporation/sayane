# Sayane v1.0.27 — Native Remaining Workstreams

## Summary

Sayane `v1.0.27` extends the native macOS daemon screen by trimming `Next Epic Workspace` down to
the workstreams that are not already covered by the priority path above it.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
reduces repeated summaries between `Start Here`, `Operator Summary Rail`, and the deeper workspace
cards.

## Highlights

- native macOS `Next Epic Workspace` now keeps only the workstreams not already surfaced in the current priority path
- when the priority path already covers the main decision surfaces, the workspace now says so directly instead of restating the same cards
- the deeper gate guide, decision assist, and evidence drill-down remain available in the same workspace block
- getting-started, handoff, and native README guidance now describe the workspace as the deeper-review layer after the priority path

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not remove phase-closure gates, decision assist, evidence drill-down, or status panels
