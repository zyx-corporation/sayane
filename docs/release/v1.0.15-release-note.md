# Sayane v1.0.15 — Resident Daemon Proof Diagnostics

## Summary

Sayane `v1.0.15` closes the first resident-daemon proof phase.

This release does **not** claim verified daemon identity, verified daemon readiness, or verified API
readiness. Instead, it adds one explicit proof-oriented diagnostic line so operators can inspect
those concerns separately without mutating state and without upgrading preview wording too early.

## Highlights

- conservative daemon identity-proof contract
- operation-class-aware daemon readiness-proof contract
- conservative API-readiness-proof contract
- shared proof status vocabulary plus downgrade-reason fields
- focused CLI proof reads plus aggregated `daemon-proof-diagnostics`
- native macOS app runbook now keeps proof-oriented CLI reads copyable from the same daemon surface
- operator handoff and app wording stay aligned with preview-oriented daemon claims

## Boundary

- daemon preview is not daemon identity proof
- daemon preview is not daemon readiness proof
- daemon preview is not API readiness proof
- proof diagnostics are read-only and non-mutating
- OS service installation, tray UX, and background supervision UX remain out of scope

## CLI

```bash
sayane app daemon-identity-proof --json
sayane app daemon-readiness-proof --operation-class bridge_health --json
sayane app daemon-api-readiness-proof --operation-class bridge_health --json
sayane app daemon-proof-diagnostics --operation-class bridge_health --json
```

## App Surface

The native macOS app keeps daemon wording preview-oriented, while the `LaunchAgent Runbook` now
surfaces the proof-oriented CLI commands directly for copy-and-inspect use.
