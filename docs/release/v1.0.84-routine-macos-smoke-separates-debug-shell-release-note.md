# v1.0.84 Routine macOS Smoke Separates Debug Shell

Date: 2026-06-30

## Summary

The normal macOS readiness smoke now separates routine operator validation from the retained
Bridge-hosted compatibility shell.

## What changed

- added `scripts/check-macos-routine-smoke.sh` for routine macOS checks
- added `--routine` to `scripts/check-resident-app-release-smoke.sh`
- updated getting-started guidance to treat HTML/UI-session smoke as maintainer/debug-only

## Why it matters

This moves everyday readiness checks closer to the actual native-first operator path while keeping
the browser compatibility shell available for explicit debug and compatibility verification.
