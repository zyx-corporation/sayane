# v1.0.78 macOS Release Zip Install Release Note

Sayane extends the current macOS native-app distribution line with a release-zip
install helper that no longer requires a repo checkout for ordinary app install
or update.

The primary supported install path still remains `curl | bash` for the CLI /
Bridge runtime. This slice only closes more of the native-shell install path on
top of that existing backend prerequisite.

## Included

- `scripts/install-macos-app-release.sh`
  - downloads `SayaneApp-latest-macos-release.zip` from GitHub Releases by default
  - supports version-pinned install with `--version`
  - supports explicit zip URL or local zip input
  - extracts and installs into `~/Applications/SayaneApp.app`
  - reuses the existing install / ad-hoc-sign / verify helpers
- `docs/install.md`
  - now documents the release-zip install path next to the repo-local build path
- `macos/SayaneApp/README.md`
  - now shows the release-zip install helper as the normal no-checkout native install/update path

## Current boundary

- this does not change the primary `curl | bash` CLI / Bridge prerequisite
- this does not create a self-contained Python bundle
- this does not claim signed pkg / notarized dmg / auto-update
- the installed native app still launches the local backend through the same
  helper/runtime boundary

## Verification used in this slice

```bash
bash -n scripts/install-macos-app-release.sh scripts/check-macos-app-release-install.sh scripts/install-macos-app.sh scripts/verify-macos-app-install.sh
bash scripts/check-macos-app-release-install.sh
swift test --package-path macos/SayaneApp
```

## Operator outcome

macOS now has two secondary native-shell install/update paths on top of the
same CLI / Bridge base runtime:

1. repo-local build/install from a checkout
2. release-zip install into `~/Applications` without a checkout
