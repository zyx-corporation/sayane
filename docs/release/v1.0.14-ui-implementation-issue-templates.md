# v1.0.14 UI Implementation Issue Templates

Status: planning_ready

## Goal

Provide GitHub-ready issue body templates for the resident app implementation sequence.

These templates follow the current resident app boundaries:

- keep `GET /app/overview` as the primary read model
- keep candidate mutation inside the existing review endpoints
- keep daemon surfaces preview/observation only
- avoid direct profile patch UI
