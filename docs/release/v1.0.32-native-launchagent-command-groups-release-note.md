# Sayane v1.0.32 — Native LaunchAgent Command Groups

## Summary

Sayane `v1.0.32` extends the native macOS daemon screen by making the `LaunchAgent` command groups
match their actual operator intent more closely.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
keeps the inspect group non-mutating and makes the recover group step through cleanup, repair, and
bootout before restart commands.

## Highlights

- native macOS `LaunchAgent` inspect group now keeps only non-mutating status and health checks
- the recover group now orders cleanup, repair, and then bootout before any restart path
- log actions now surface paths before tail commands so the operator sees the files before following them
- getting-started, handoff, and native README guidance now describe the LaunchAgent command groups as explicit inspect / recover / start / log boundaries

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not change the underlying CLI commands; it only groups and orders them more clearly in the native UI
