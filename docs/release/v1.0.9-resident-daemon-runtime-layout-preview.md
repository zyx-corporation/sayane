## Added

### Runtime layout contract

The app layer now defines a plan-only runtime layout for future resident daemon work.

Default planned directories are:

```text
.sayane/run/pid
.sayane/run/lock
.sayane/run/socket
.sayane/run/log
.sayane/run/tmp
.sayane/run/state
```

The layout validates child paths under the runtime root.

### Runtime layout CLI preview

New command:

```bash
sayane app daemon-runtime-layout --json
```

Optional runtime root override:

```bash
sayane app daemon-runtime-layout --runtime-root /path/to/runtime --json
```

The JSON payload includes:

```text
runtime_root
pid_dir
lock_dir
socket_dir
log_dir
temp_dir
state_dir
creates_directories
writes_files
is_filesystem_mutation
kind
preview_only
```

## Changed

### Version reporting

`sayane --version` now reads installed package metadata.

This prevents the CLI version from drifting away from `pyproject.toml` and `pip show sayane`.

## Safety boundaries

This release remains preview-only and non-mutating.

The runtime layout preview does not:

- create directories
- write PID files
- write lock files
- create sockets
- write logs
- create temp files
- persist runtime state files
- acquire locks
- clean stale runtime files
- start, stop, or restart daemon processes
- install OS services

The command explicitly reports:

```text
creates_directories: false
writes_files: false
is_filesystem_mutation: false
preview_only: true
```

## RDE Delta-M Review

### Preserved

- Resident daemon runtime layout remains a contract, not an active daemon.
- No hidden filesystem mutation is introduced.
- Version metadata has a single source of truth.

### Supplemented

- Operators and future UI surfaces can now inspect planned runtime directories.
- The release chain has an explicit CI success record.

### Deviation risk

Do not treat runtime layout preview output as proof that runtime directories exist.

Do not treat runtime layout preview output as process state.

Do not infer daemon readiness from layout metadata alone.
