# Sayane v1.0.19 — Bridge Operator Drill-Down

## Summary

Sayane `v1.0.19` extends the current operator read stack by making the bridge-hosted daemon shell
link directly to dedicated packaging, service-target, service-boundary, supervision, and
recovery read surfaces.

This still does **not** close the packaging and supervision phase. It only makes the same
read-only operator evidence easier to inspect from the browser surface without falling back to one
large aggregated payload.

## Highlights

- bridge now exposes dedicated app-facing daemon drill-down endpoints for packaging, service targets, service boundary, supervision, and recovery
- bridge UI session now exposes matching `/app/ui-state/daemon-*` drill-down endpoints
- bridge-hosted daemon shell now shows an operator workspace and evidence drill-down panel
- launcher and getting-started now call out the new browser/token-backed drill-down paths

## Boundary

- this release does not add service install / enable / disable / remove UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not claim daemon proof or phase closure

## Recommended reads

```bash
curl -s -H "Authorization: Bearer $(cat ~/.sayane/bridge.token)" http://127.0.0.1:38741/app/daemon-packaging-status
curl -s -H "Authorization: Bearer $(cat ~/.sayane/bridge.token)" http://127.0.0.1:38741/app/daemon-service-targets-status
sayane app daemon-service-control-boundary --json
sayane app daemon-supervision-status --json
sayane app daemon-recovery-consent-status --json
```
