# Sayane v1.0.8 Resident Daemon Identity Preview

## Release scope

This release packages resident daemon process identity preview work.

It is an identity and CLI preview release, not a production daemon release.

## Included issues

```text
#191 Resident daemon process identity and lockfile contract
#192 Resident daemon identity CLI preview
```

## Highlights

- Plan-only resident daemon process identity contract.
- Planned PID, lock, and socket paths.
- Runtime-local path validation for PID, lock, and socket paths.
- Read-only daemon identity CLI:
  - `sayane app daemon-identity --json`
- Runtime directory override:
  - `sayane app daemon-identity --runtime-dir /path/to/runtime --json`

## Safety boundaries

The release deliberately does not provide:

- PID file writes
- lock acquisition
- socket creation
- process control
- process registry semantics
- stale lock cleanup
- OS service integration

## Current identity relationship

The identity preview command reports planned paths only.

```text
writes_files: false
acquires_lock: false
is_process_control: false
preview_only: true
```

## RDE Delta-M Review

### Preserved

No hidden resident daemon process is introduced.

Lifecycle and operation commands remain plan-only.

### Transformed

Daemon identity moves from an implicit future concern to a testable process identity contract.

### Supplemented

Operators and future UI code can inspect planned PID, lock, and socket paths.

### Unresolved

PID writes, lock acquisition, stale lock handling, process registry behavior, and OS service integration remain future work.

### Deviation Risk

Do not present v1.0.8 as a production daemon release.

Do not treat daemon identity preview as proof that a daemon is running.
