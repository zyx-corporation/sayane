# v1.0.44 Native Daemon Handoff Diagnostic Paths Release Note

## Summary

The native macOS daemon handoff export now includes component identity plus token/log paths for faster first-pass troubleshooting.

## What changed

- added `Token File` and `Log File` lines to the exported handoff note
- added `Component` when the Bridge health payload exposes it
- kept the export plain-text, local-first, and read-first

## Operator impact

- saved handoff notes now include the exact local files needed for reconnect and log-first debugging
- the exported artifact is more useful when handed off to someone who did not generate it
- first-pass troubleshooting no longer requires reopening the app just to recover token/log locations

## Validation

- `swift build --package-path macos/SayaneApp`
- focused `swift test --package-path macos/SayaneApp --filter ...` still intermittently stalls after test bundle link through `swiftpm-xctest-helper` in this environment
