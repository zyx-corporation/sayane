# Sayane v1.0.25 — Native Start Here Navigation

## Summary

Sayane `v1.0.25` extends the native macOS daemon screen by turning the `Start Here` cards into
direct section navigation instead of passive summary-only cards.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
reduces the gap between “what should I inspect first?” and “which daemon section should I open?”.

## Highlights

- native macOS daemon `Start Here` cards now jump directly to the matching daemon section
- the next-action card now routes commands like preflight, service-control, supervision, recovery, and packaging to the most relevant section
- LaunchAgent status and runbook cards now expand and scroll to their sections instead of acting as static summaries
- getting-started, handoff, and native README guidance now describe `Start Here` as the first navigation layer, not only a status layer

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not remove the deeper operator summary rail, workspace, gate guide, or drill-down sections
