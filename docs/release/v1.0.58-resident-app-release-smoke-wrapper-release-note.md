# v1.0.58 Resident App Release Smoke Wrapper Release Note

## Summary

- added `scripts/check-resident-app-release-smoke.sh` as one entrypoint for the
  resident app release-smoke path
- runs bearer-backed API smoke and compatibility-shell UI session smoke by default
- keeps native macOS preview smoke opt-in via `--with-native`

## Why

- we now have stable direct smoke paths for app-facing JSON and Bridge-hosted UI
  session boundaries
- a single wrapper makes release verification easier than remembering multiple
  script names and flag combinations

## Validation

- `bash -n scripts/check-resident-app-release-smoke.sh`
- `bash scripts/check-resident-app-release-smoke.sh --start`
