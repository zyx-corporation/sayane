# v1.0.14 Real UI E2E and Regression Implementation Skeleton

Status: implementation_ready

## Goal

Turn the real UI E2E/regression design note into an implementation-ready skeleton for the
production phase.

## Expected file touch areas

### 1. Harness entry

- UI startup
- auth/session bootstrap
- teardown and cleanup

### 2. Main path scenarios

- capture path
- queue/detail/diff path
- action-flow path
- daemon panel path

### 3. Assertion strategy

- stable semantic assertions first
- bounded visual/interaction assertions where operator usability depends on them

### 4. CI integration

- runnable production-path regression entry
- clear failure reporting

## Acceptance criteria

- release confidence no longer depends on manual-only walkthroughs
- the main operator paths are covered by GUI-level regression
- failures remain diagnosable against stable contracts and screen semantics

## Test plan

- local harness smoke
- CI regression path run
- targeted failure-mode checks for the main operator paths
