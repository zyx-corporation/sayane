# v1.0.60 Resident App Smoke Venv CLI Preference Release Note

## Summary

- changed resident app smoke scripts to prefer the repo-local
  `.venv/bin/sayane` before any user-local shim
- keeps smoke startup aligned with the current checkout's installed package
- avoids detached startup drift from older PATH-level wrappers

## Why

- the repo-local CLI is the most direct match for the code under test
- user-local shims can point at a different environment or behave differently
  under detached smoke startup

## Validation

- `bash scripts/check-resident-app-api-surfaces.sh --start`
- `bash scripts/check-resident-app-ui-session.sh --start`
- `bash scripts/check-resident-app-release-smoke.sh --start`
