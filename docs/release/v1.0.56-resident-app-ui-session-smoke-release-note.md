# v1.0.56 Resident App UI Session Smoke Release Note

## Summary

- added `scripts/check-resident-app-ui-session.sh` for a direct Bridge-hosted UI
  session smoke path
- validates bootstrap, cookie-backed HTML/JSON reads, and logout invalidation
- gives us a non-pytest regression path when `tests/test_bridge_api.py` is slow
  to iterate in the current environment

## Why

- the resident app UI now depends on dedicated UI session cookies, locale
  persistence, and separate bearer-vs-cookie boundaries
- those boundaries are important enough to verify outside full pytest runs

## Validation

- `bash -n scripts/check-resident-app-ui-session.sh`
