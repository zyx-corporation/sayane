# Sayane v1.0.6 Resident Foundation

## Release scope

This release closes the front-loaded ADR 0007 resident app and Local Vault foundation work.

It is a foundation release, not a production daemon release.

## Included issues


## Highlights

- RepositoryBundle boundary for Candidate, ReviewDecision, Lineage, profile context, and project context.
- Repository-backed ReviewDecision seam.
- SQLite-backed Local Vault MVP persistence path.
- ResidentAppService and ResidentRuntime boundaries.
- Clipboard capture through resident app CLI as pending Candidate input.
- Resident app serve delegation to existing Bridge command.
- Resident review queue and MCP preview skeletons.
- Read-only preview CLI commands:
  - `sayane app review-queue --json`
  - `sayane app mcp-preview --json`
- Explicit resident repository backend selection.
- Capability issuer policy metadata and surface-scoped local issuance.

## Safety boundaries

The release deliberately does not provide:

- production resident daemon lifecycle
- OS service installation
- OS keychain integration
- durable credential storage
- network authentication
- production token signing
- pro backend implementation

## RDE Delta-M Review

### Preserved

Sayane remains local-first, Candidate-mediated, lineage-aware, and review-centered.

CLI, Bridge, MCP, and future UI paths are kept behind repository and usecase boundaries.

### Transformed

The architecture moves from separate process-local state toward a resident-compatible shared state foundation.

### Supplemented

Runtime backend selection, resident preview payloads, and capability policy metadata are now explicit and testable.

### Unresolved

The resident daemon lifecycle, production unlock-session binding, durable credentials, and pro backend remain future work.

### Deviation Risk

Do not present v1.0.6 as a production daemon release.

Do not treat `sqlite_test_local_vault` or local capability tokens as production security.
