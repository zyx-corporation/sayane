# v1.0.79 Native Debug Compatibility Disclosure Retirement Release Note

Sayane continues the macOS-side `/app/ui*` retirement by removing routine
browser-opening actions from the native operator flow and exposing the retained
compatibility surface only when explicit diagnostics or handoff work needs it.

## Included

- native diagnostics now hide debug compatibility tools during ordinary healthy
  or simple reconnect flows
- native diagnostics keep the retained compatibility URL as a copy-only action
  when explicit failure or session-debug work needs it
- routine operator docs now describe `/app/ui` as an explicit fallback instead
  of a nearby day-to-day recovery action

## Current boundary

- `/app/ui*` still exists as a maintainer/debug compatibility surface
- native handoff/export may still include the compatibility URL
- maintainers can still force visibility with `SAYANE_SHOW_DEBUG_COMPATIBILITY_TOOLS=1`

swift test --package-path macos/SayaneApp
