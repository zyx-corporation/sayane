# v1.0.65 Native Primary UI and Debug-Only Compatibility Release Note

## Summary

- moved the resident app direction more explicitly to **native macOS first**
  while treating `/app/ui` as a debug-only compatibility surface
- reshaped native Bridge / queue / daemon / error surfaces so recovery,
  next-command guidance, and evidence drill-down stay action-first
- hardened local launcher and smoke scripts so they fail closed unless they
  find a compatible Python runtime with the required Sayane dependencies
- aligned local docs around `uv run --extra dev ...` as the most reliable
  validation path when user-local `pytest` shims are broken
- fixed package build exclusions so local release artifacts no longer capture
  `macos/SayaneApp/.build` output inside the source distribution

## Why

- native macOS is now the product-direction UI, so docs and operator recovery
  flow should stop presenting browser bootstrap as the default path
- resident app recovery is more useful when the current gate, startup command,
  and next daemon/read actions stay visible in the same place as the reconnect
  controls
- local launch reliability should not depend on whichever ambient Python or
  shell shim happens to come first on a workstation

## Highlights

- ADR `0014` records the decision that `/app/ui` remains available only as a
  debug / smoke / handoff compatibility surface
- the native Bridge status panel now exposes:
  - suggested recovery action
  - startup command
  - current gate
  - next daemon command and reason
  - next read surface
- queue filters and daemon navigation badges now wrap inside the native window
  instead of forcing horizontal chip scrolling at narrower widths
- launcher / smoke scripts now refuse incompatible Python runtimes early and
  point the operator toward `uv run --extra dev ...` or a prepared `.venv`
- package build now explicitly excludes local build artifacts such as
  `macos/SayaneApp/.build/**`, keeping the sdist at release-appropriate size

## Boundary

- this release does not remove `/app/ui` yet
- this release does not re-open extension work
- this release does not change the current bearer-backed `/app/...` contract
- this release does not add service-mutating native controls beyond the current
  CLI-first local boundary
