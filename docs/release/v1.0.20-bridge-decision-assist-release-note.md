# Sayane v1.0.20 — Bridge Decision Assist

## Summary

Sayane `v1.0.20` extends the bridge-hosted daemon shell again by adding decision-assist cards for
the current service-control, recovery, supervision, and LaunchAgent review path.

This still does **not** close the packaging and supervision phase. It keeps the surface read-first,
but now makes the next command easier to choose from the same browser workspace.

## Highlights

- bridge-hosted daemon shell now shows decision-assist cards beside operator workspace and evidence drill-down
- decision assist now points directly to the next service-control, recovery, supervision, and LaunchAgent inspection command
- bridge-hosted shell now links those assist cards to the matching `/app/ui-state/daemon-*` read surfaces
- getting-started and handoff docs now call out decision assist as part of the current operator read stack

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not claim daemon proof, readiness proof, or phase closure
