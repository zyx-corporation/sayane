# v1.0.69 P2 Working-Store Boundary Closure Release Note

## Summary

Sayane `v1.0.69` closes the remaining post-migration decision for Issue `P2
Sensitive working-store migration`.

After this slice, transitional FileSystem working stores are explicitly retained
only as debug / legacy / import-fixture compatibility paths. They are no longer
an open-ended “temporary default” for future app-facing sensitive-state work.

## Included work

- ADR `0017` fixes the retained role of transitional FileSystem working stores
- roadmap status now treats `P2` as closed rather than “complete except one open decision”
- release docs now state that Local Vault / repository-backed persistence is the
  forward path for `Candidate`, `ReviewDecision`, and `Lineage`

## Boundary retained

- FileSystem working stores still exist for debug, smoke, fixture, and legacy
  compatibility paths
- `/app/ui*` remains a separate debug-only compatibility decision and is not
  retired by this release
- external integration redesign remains Issue `P3`

## Next line

- `P3 External integration re-design`
- `P6 /app/ui* legacy retirement`
- cross-platform packaging completion for Linux and Windows
