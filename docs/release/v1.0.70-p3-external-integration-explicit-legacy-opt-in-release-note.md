# v1.0.70 P3 External Integration Explicit Legacy Opt-In Release Note

## Summary

Sayane `v1.0.70` starts the first implementation slice of Issue `P3 External
integration re-design`.

Obsidian import/export and Git commit remain available, but they are no longer
casual mutating defaults. They now require explicit compatibility confirmation.

## Included work

- `sayane storage import` requires `--legacy-compatible` for mutating use
- `sayane storage export` requires `--legacy-compatible`
- `sayane storage commit` requires `--legacy-compatible`
- `sayane init` no longer creates a Git repository or initial commit implicitly
- dry-run inspection remains available without the same confirmation
- exported compatibility artifacts now include metadata and notice files that
  state the export is derived context, not canonical profile state
- export metadata now also carries redacted-path and retention guidance fields
- export target subdir now rejects hidden, reserved, and path-escaping targets
- import can now be bounded to a safe `--source-subdir` under the external
  vault root

## Boundary retained

- these paths still exist as compatibility workflows
- they are still not the primary Local Vault-sensitive-state path
- this release does not define the final supported external sync/export model

## Next remaining P3 work

- define retention and redaction rules for external artifacts
- define which external workflows become supported versus permanently legacy
- define whether preview-only package import later persists review candidates
