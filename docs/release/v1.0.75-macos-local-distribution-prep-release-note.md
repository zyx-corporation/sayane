# v1.0.75 macOS Local Distribution Prep Release Note

Sayane extends the current macOS native-app line with an operator-usable local
distribution and release-preparation workflow.

This slice does **not** claim signed or notarized shipping artifacts yet.
Instead, it makes the current local-first packaging line reproducible and
easier to verify before maintainer credentials are introduced.

The primary install path still remains `curl | bash` for CLI / Bridge. The
native app packaging line in this note is a secondary operator path layered on
top of that base runtime.

## Included

- `scripts/build-macos-app-bundle.sh`
  - builds `dist/macos/SayaneApp.app`
  - can also emit a zip archive for handoff or later notarization input
  - now also emits checksum and manifest metadata plus a stable latest-zip name
- `scripts/install-macos-app.sh`
  - installs the current bundle into `~/Applications/SayaneApp.app`
- `scripts/uninstall-macos-app.sh`
  - removes the local install cleanly
- `scripts/verify-macos-app-install.sh`
  - verifies installed bundle structure and current signing state
- `scripts/sign-macos-app.sh`
  - prepares the real `codesign` flow and reports pre-sign state cleanly
- `scripts/notarize-macos-app.sh`
  - prepares the real `notarytool` flow without embedding maintainer secrets
- `scripts/prepare-macos-release-artifacts.sh`
  - bundles build + zip + pre-sign verification into one local release-prep path

## Current boundary

- the native app still expects the local CLI / Bridge runtime
- the produced app is intentionally allowed to remain in `unsigned` pre-sign
  state until Developer ID credentials are available
- signed pkg / notarized dmg remain future maintainer steps rather than part of
  the current Community release artifact

## Verification used in this slice

```bash
bash scripts/build-macos-app-bundle.sh --zip
bash scripts/install-macos-app.sh --no-open --no-build
bash scripts/verify-macos-app-install.sh
swift test --package-path macos/SayaneApp
```

## Operator outcome

macOS now has a clearer local distribution path:

1. build bundle
2. install into `~/Applications`
3. verify structure and pre-sign state
4. hand off the same bundle/zip to maintainer signing and notarization later

The build output now also includes a stable `SayaneApp-latest-macos-release.zip`
name plus checksum/manifest files so local handoff and release-prep do not need
to guess the versioned artifact name first.
