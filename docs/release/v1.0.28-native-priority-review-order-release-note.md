# Sayane v1.0.28 — Native Priority Review Order

## Summary

Sayane `v1.0.28` extends the native macOS daemon screen by aligning the deeper review blocks with
the same priority path already used by `Start Here`, `Operator Summary Rail`, and `Section Navigator`.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
keeps the reading order stable when the operator moves from top-level orientation into deeper review.

## Highlights

- native macOS deeper review blocks now reorder `Phase Closure Gates`, `Decision Assist`, `Evidence Drill-Down`, and `Remaining Workstreams` by the same current priority path
- cards inside those blocks now also sort by the same anchor order instead of drifting into a generic static order
- the current priority path now survives from the top of the daemon screen into the deeper review area without a separate mental reset
- getting-started, handoff, and native README guidance now describe the deeper review area as following the same priority path

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not remove any existing deeper review surface; it only changes their order to match the current path
