# Sayane v1.0.26 — Native Priority Section Navigation

## Summary

Sayane `v1.0.26` extends the native macOS daemon screen by splitting the section navigator into
priority sections and remaining sections.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
keeps the top-of-screen operator path aligned across `Start Here`, `Operator Summary Rail`, and the
section navigator itself.

## Highlights

- native macOS daemon `Section Navigator` now shows `Priority Sections` first and `Other Sections` separately
- priority sections are derived from the same current path already surfaced in `Start Here` and `Operator Summary Rail`
- the operator workspace remains reachable from the priority group so the bridge between top-level orientation and deeper review stays short
- getting-started, handoff, and native README guidance now describe the navigator as a continuation of the same priority path

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not remove the deeper operator workspace, gate guide, drill-down, or status panels
