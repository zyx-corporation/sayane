# v1.0.59 Resident App Smoke CLI Bridge Start Release Note

## Summary

- switched the resident app API/UI smoke scripts to start the Bridge via
  `sayane serve` instead of `python -m sayane.cli.main serve`
- keeps smoke startup aligned with the user-facing CLI path
- avoids detached module-launch drift between environments

## Why

- the CLI entrypoint is the most stable operator-facing start path in this
  checkout
- using the same start command in smoke scripts reduces startup surprises during
  release verification

## Validation

- `bash scripts/check-resident-app-api-surfaces.sh --start`
- `bash scripts/check-resident-app-ui-session.sh --start`
