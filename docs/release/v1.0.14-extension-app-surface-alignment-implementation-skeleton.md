# v1.0.14 Extension and App Surface Alignment Implementation Skeleton

Status: implemented

## Goal

Turn the extension/app surface-alignment design note into an implementation-ready skeleton for the
production phase.

## Expected file touch areas

### 1. Shared contract documentation

- app-facing endpoints
- screen-state payload semantics
- shared boundary wording

### 2. Shared wiring points

- auth/session handling where reuse is intended
- screen-state readers and adapters

### 3. Deliberate divergence documentation

- compatibility-only extension behavior
- resident-app-primary behavior

### 4. Drift checks

- focused tests or assertions where contract drift would be costly

## Acceptance criteria

- primary vs compatibility surface roles are explicit
- shared semantics are documented and testable
- accidental auth or review-semantics drift is reduced by design

## Implemented outcome

- `src/sayane/app/app_contract.py` exposes `surface_roles` and `shared_semantics`
- `extension/src/types.ts` mirrors those contract fields for extension-side compatibility consumers
- `tests/test_resident_app_contract.py` verifies the primary-vs-compatibility split and the shared-vs-separate semantics matrix

## Test plan

- shared contract drift checks
- screen-state reader drift checks
- explicit documentation review for shared vs separate behavior
