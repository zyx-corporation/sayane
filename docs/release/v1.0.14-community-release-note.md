# Sayane Community v1.0.14

Resident App Release Reliability

## Summary

Sayane v1.0.14 packages the current resident app release-readiness line.

This release adds or stabilizes:

- guarded resident app Bridge startup when optional native `watchfiles` wheels
  are broken
- aligned browser and native macOS smoke around the same local Bridge launcher
- one-command resident app release smoke across API surface, UI session, and
  native preview
- isolated local package metadata verification that does not depend on a broken
  long-lived development virtualenv

## Commands

```bash
bash scripts/check-resident-app-release-smoke.sh --start --with-native
bash scripts/build-wheel.sh
```

## Operator impact

- the current local release path is now practical as one browser/native smoke
  command plus one package build command
- local package verification no longer depends on the maintainer's long-lived
  `twine` / `nh3` environment state
- current resident app startup remains local-only and does not change the
  existing non-MVP daemon or OS service boundaries

## Safety boundary retained

- No production resident daemon runtime.
- No daemon process identity proof by startup alone.
- No daemon readiness or API readiness proof by startup alone.
- No direct profile patch UI.
- No OS service installation or background supervision closure in this package
  release.
