# Sayane v1.0.35 — Native LaunchAgent Detail Split

## Summary

Sayane `v1.0.35` refines the native macOS `LaunchAgent` section so the compact `LaunchAgent Focus`
block stays as the first-glance summary, while the lower groups now act as detail verification
surfaces instead of repeating the same summary content.

This still does **not** close the packaging and supervision phase. It remains read-first and keeps
the current CLI-first mutation boundary unchanged.

## Highlights

- the native `LaunchAgent Focus` block remains the first operator scan point
- the lower `Current State Details` group now focuses on plist path and loaded-status verification
- the lower `Recovery Preview Details` group now focuses on diagnostic breakdowns and cautions
- getting-started, handoff, and native README guidance now describe the lower groups as detail
  follow-through surfaces rather than repeated summaries

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change any LaunchAgent command ordering or command payload
- this release does not change the current `cli_first_local_bridge` operator path
