# v1.0.67 Local Vault Foundation P1 Closure Release Note

## Summary

Sayane `v1.0.67` closes Issue `P1 Local Vault foundation`.

This line completes the first production-completion security slice for the resident app by making
Local Vault status, scoped unlock sessions, and unlock-gated write behavior explicit across CLI,
Bridge, and the native macOS app.

## Included work

- explicit Local Vault runtime selection for:
  - lower-assurance development SQLite + passphrase
  - explicit macOS Keychain + SQLite
  - explicit test-only SQLite runtime
- `PlatformKeychainProvider` / `KeyManager` / `CryptoProvider` seams
- scoped unlock policy presets for `normal`, `sensitive`, and `deep_private`
- process-local unlock session manager with timeout metadata
- resident app `vault-status` and `vault-session` read surfaces
- resident app unlock open / lock write surfaces
- native macOS Home visibility and controls for Local Vault status and unlock sessions
- fail-closed write gating when Local Vault is active and no valid unlock session exists

## Operator impact

- the native macOS app can now show whether the resident runtime is using a Local Vault backend
- operators can see active unlock-session levels before attempting capture or review writes
- clipboard capture, candidate review, and candidate revision now fail closed when a Local Vault
  backend is active but the required unlock session is missing
- Bridge and native-app troubleshooting now have explicit Local Vault status/session reads

## Current boundary retained

- no cross-platform production-default keychain/runtime selection yet
- Local Vault remains explicit opt-in rather than implicit production default
- remaining FileSystem-backed working stores are still transitional until Issue `P2` migration
- `/app/ui*` remains maintainer/debug compatibility, not the primary operator surface
- extension retirement remains a separate later issue

## Next recommended issue

Issue `P2 Sensitive working-store migration`

- move `Candidate`, `ReviewDecision`, and `Lineage` working data fully onto vault-aware storage
- remove remaining high-sensitivity dependence on transitional FileSystem local working stores
