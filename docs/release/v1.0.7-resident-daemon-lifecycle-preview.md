# Sayane v1.0.7 Resident Daemon Lifecycle Preview

## Release scope

This release packages the resident daemon lifecycle preview track.

It is a lifecycle and CLI preview release, not a production daemon release.

## Included issues

```text
#187 Resident daemon lifecycle design
#188 Resident daemon CLI lifecycle commands
#189 Resident daemon start/stop plan commands
```

## Highlights

- Resident daemon lifecycle contract model.
- Explicit lifecycle states:
  - `stopped`
  - `starting`
  - `running`
  - `stopping`
  - `failed`
- Explicit ownership modes:
  - `bridge_delegation`
  - `resident_server_reserved`
- Localhost-only bind validation.
- Read-only lifecycle CLI commands:
  - `sayane app daemon-status --json`
  - `sayane app daemon-plan --json`
- Plan-only operation CLI commands:
  - `sayane app daemon-start-plan --json`
  - `sayane app daemon-stop-plan --json`
  - `sayane app daemon-restart-plan --json`

## Safety boundaries

The release deliberately does not provide:

- production resident daemon process control
- OS service installation
- systemd, launchd, or Windows Service support
- PID file management
- durable credentials
- vault unlock-session binding
- network-exposed resident API

## Current serving relationship

The active serving path remains Bridge delegation:

```text
sayane app serve
  -> sayane serve
```

Daemon plan commands only describe future lifecycle operations.

## RDE Delta-M Review

### Preserved

Sayane keeps Bridge as the active serving path.

No hidden background process is introduced.

Local-only bind policy remains enforced.

### Transformed

Daemon lifecycle moves from architecture note to testable lifecycle and CLI contract.

### Supplemented

Operators and future UI code get stable JSON lifecycle and operation-plan payloads.

### Unresolved

Real daemon start/stop, OS service integration, durable credentials, and vault unlock-session binding remain future work.

### Deviation Risk

Do not present v1.0.7 as a production daemon release.

Do not use operation-plan commands as if they controlled processes.
