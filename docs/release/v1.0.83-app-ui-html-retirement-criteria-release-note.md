# v1.0.83 App UI HTML Retirement Criteria

Date: 2026-06-30

## Summary

The remaining server-rendered `/app/ui` HTML pages now have explicit retirement criteria.

## What changed

- accepted ADR 0029 for the server-rendered `/app/ui` HTML layer
- clarified that HTML pages are removable presentation wrappers, not permanent transport contracts
- left `/app/ui-state/*` and `/app/ui-action/*` intact as maintainer-only debug seams per ADR 0028

## Why it matters

This closes the last major ambiguity in the macOS-native-first browser-retirement story:

- cookie-backed JSON debug seams remain available
- server-rendered HTML can retire later when docs, smoke, and maintainer workflows no longer need
  it

## Follow-up

- keep shrinking routine references to `/app/ui`
- remove the HTML pages only after native/CLI/debug-seam parity is demonstrated
