# v1.0.14 Resident App Surface Handoff

Status: handoff_superseded_by_operator_ready_shell

## Summary

The current resident app phase now has an app-facing read/write surface suitable for UI
implementation handoff.

This handoff has now been realized on the Bridge-hosted resident app local shell path. For the
current operator-facing entrypoint and troubleshooting line, prefer
`docs/release/v1.0.14-resident-app-operator-handoff.md`.

The surface includes:

- contract metadata
- aggregate overview payload
- focused daemon overview payload
- candidate queue/detail/diff reads
- clipboard capture
- candidate evaluate/revise/approve/reject actions

## Preferred implementation order

UI work should start from:

```text
GET /app/contract
GET /app/overview
```

Then add:

```text
GET /app/candidates
GET /app/candidates/{id}
GET /app/candidates/{id}/diff
```

Finally wire actions:

```text
POST /app/capture-clipboard
POST /app/candidates/{id}/evaluate
POST /app/candidates/{id}/revise
POST /app/candidates/{id}/approve
POST /app/candidates/{id}/reject
```

## What is complete for this phase

- app-facing contract payload exists
- app-facing aggregate overview exists
- app-facing review flow exists
- app-facing daemon overview exists
- candidate flow remains inside review/approval boundary

## What remains out of scope

- final GUI framework
- system tray UI
- direct profile patch surface
- production daemon identity/readiness proof
- OS service integration
- durable local auth or keychain-backed credentials
