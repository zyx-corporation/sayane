# v1.0.62 Native Smoke Launcher Alignment Release Note

## Summary

- aligned `scripts/check-macos-app-preview.sh` and `scripts/run-macos-app-preview.sh`
  with the same `run-app-local.sh` Bridge startup path used by the resident app shell
- taught macOS smoke and launcher scripts to stop stale non-listening `serve`
  processes before restart
- raised the native smoke Swift test timeout to match the current package-wide
  `swift test --disable-xctest` execution time

## Why

- native preview validation should exercise the same Bridge startup path that
  operators and resident app smoke already use
- stale detached `serve` processes created flaky startup behavior across native
  smoke and preview launch
- the full Swift package test suite is fast enough for smoke validation, but no
  longer fits the previous 45-second timeout budget reliably

## Validation

- `bash scripts/check-macos-app-preview.sh --no-build --no-tests`
- `bash scripts/check-macos-app-preview.sh`
