# Sayane v1.0.78 — Terminal-First Bridge Launch

## Summary

Sayane tightens the current macOS local launch path so the native preview and
Bridge helper both prefer a more operator-visible Bridge startup model.

## Highlights

- `scripts/run-app-local.sh` now supports explicit `--foreground`,
  `--background`, and `--terminal` Bridge start modes
- on macOS, `scripts/run-app-local.sh` now prefers `--terminal` behavior in
  auto mode instead of relying on an in-process detached background launch
- the local launcher now reuses an already healthy Bridge instead of stopping it
- `scripts/run-macos-app-preview.sh` now exposes matching
  `--bridge-terminal` / `--bridge-background` / `--bridge-foreground` options
- native preview summaries now print Bridge health, Bridge log, and the chosen
  Bridge start mode directly

## Boundary

- this release does not change the Bridge token or UI-session contract
- this release does not remove the maintainer/debug compatibility shell
- this release does not package the native app as a self-contained shipped app

## Recommended reads

```bash
./scripts/run-app-local.sh
./scripts/run-app-local.sh --terminal --no-open
./scripts/run-macos-app-preview.sh --bridge-terminal --no-build
./scripts/run-macos-app-preview.sh --bridge-background --no-build
```
