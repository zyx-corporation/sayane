Status: planning_ready

## Goal

Provide a no-decision execution checklist for creating the resident app implementation issues in GitHub.

This checklist assumes the companion title/body docs are already accepted and the next step is issue creation work.

## Suggested GitHub workflow

For each issue:

1. copy the title from `v1.0.14-ui-implementation-issue-title-list.md`
2. copy labels from `v1.0.14-ui-implementation-github-issue-batch.md`
3. copy the body from `v1.0.14-ui-implementation-github-issue-batch.md`
4. create the issue
5. paste the created issue URL back into a local tracking note or project board

## Post-creation checks

- [ ] all 14 issues exist
- [ ] labels are applied correctly
- [ ] MVP issues are ordered before production issues in the project board/backlog
- [ ] daemon-related issues keep daemon boundary wording intact
- [ ] auth-related issues do not imply current bootstrap auth is production-ready

## Optional `gh` command pattern

If issue creation is done via GitHub CLI, the following pattern is the intended shape:

```text
gh issue create --title "<title>" --body-file <body-file> --label "<label1>" --label "<label2>"
```

This document does not require one exact command per issue; use the batch doc as the source of truth for title, labels, and body text.
