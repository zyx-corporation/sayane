# Sayane v1.0.36 — Native Local Launcher and Smoke Reliability

## Summary

Sayane `v1.0.36` hardens the local native-app launch and smoke-check scripts so the resident app
bootstrap path is easier to reuse in Chrome and the smoke check no longer depends on the unstable
full Swift package test suite.

## Highlights

- `scripts/run-app-local.sh` now prefers `Google Chrome` for the bootstrap URL when it is installed
- `scripts/check-macos-app-preview.sh` now supports `--test-filter <name>` and uses a fast filtered
  Swift test by default during smoke validation
- curl-based smoke steps now use an explicit timeout so session/bootstrap failures return promptly
- getting-started, handoff, and native README guidance now reflect the Chrome-first bootstrap path
  and the filtered smoke-test workflow

## Boundary

- this release does not change the resident app routing or token/session contract
- this release does not claim that the full `swift test --package-path macos/SayaneApp` suite is
  stable yet
- this release does not add service-control mutation to the native app

## Recommended reads

```bash
./scripts/run-app-local.sh --no-open
./scripts/check-macos-app-preview.sh --no-build --no-start
./scripts/check-macos-app-preview.sh --test-filter appStringsLocalizeSummaryCardLabels --no-start
open -a "Google Chrome" "http://127.0.0.1:38741/app/ui?bootstrap_token=$(cat ~/.sayane/bridge.token)"
```
