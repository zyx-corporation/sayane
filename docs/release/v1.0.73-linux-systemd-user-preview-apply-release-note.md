# Sayane v1.0.73 — Linux systemd --user Preview/Apply Slice

## Summary

Sayane adds the first concrete Linux operator-packaging slice for the resident
daemon: an explicit `systemd --user` preview/apply/status path.

This closes the old "contract only" gap for Linux unit generation while keeping
full service lifecycle automation out of scope for now.

## What changed

- added `sayane app daemon-systemd-user-preview --json`
- added `sayane app daemon-systemd-user-apply --json`
- added `sayane app daemon-systemd-user-status --json`
- upgraded Linux service-target metadata from contract-only to explicit
  preview/apply unit-write support
- updated packaging/control status surfaces so Linux now reports a concrete
  post-MVP local operator path instead of a pure placeholder

## Boundaries

This release does not yet:

- automate `systemctl --user enable/disable/update/remove`
- ship Linux desktop packaging
- change the primary macOS-native UI direction
- re-open the extension line
