# Sayane v1.0.31 — Native LaunchAgent Command Deck Order

## Summary

Sayane `v1.0.31` extends the native macOS daemon screen by reordering the `LaunchAgent` command
deck to follow the same inspect-first flow as the current-state and Runbook sections.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
places recovery actions before start actions so the operator can clear stale state before trying to
restart.

## Highlights

- native macOS `LaunchAgent` command deck now orders actions as inspect, recover, start, and log
- recovery actions now appear before start actions so cleanup, repair, and bootout paths come before bootstrap or kickstart
- the command deck now matches the same current-state and Runbook read path used elsewhere in the LaunchAgent surface
- getting-started, handoff, and native README guidance now describe the command deck as an inspect-first operator flow

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not change the underlying CLI commands; it only changes their display order in the native UI
