# Sayane v1.0.29 — Native LaunchAgent Current State

## Summary

Sayane `v1.0.29` extends the native macOS daemon screen by keeping LaunchAgent current state and
recovery previews together in the same read-first section.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
keeps macOS-specific operator checks closer together instead of splitting current state and preview
diagnostics across separate daemon sections.

## Highlights

- native macOS `LaunchAgent` section now shows current state and recovery previews together
- loaded status, return code, runtime-init, cleanup, and repair previews now read in one macOS-focused block
- the separate daemon-wide recovery preview section is removed in favor of the more specific LaunchAgent context
- getting-started, handoff, and native README guidance now describe LaunchAgent checks as one continuous read-first path

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not change the underlying preview commands; it only groups their read surfaces more coherently in the native UI
