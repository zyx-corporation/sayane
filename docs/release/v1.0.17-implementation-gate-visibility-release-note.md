# Sayane v1.0.17 — Implementation Gate Visibility

## Summary

Sayane `v1.0.17` extends the current operator observation line with one more explicit read surface:
the resident-daemon implementation gate preflight.

This release still does **not** begin production daemon implementation. It only makes the existing
schema-only implementation gate preview visible from the same operator paths already used for
packaging, supervision, recovery, and proof diagnostics.

## Highlights

- native macOS app `LaunchAgent Runbook` now includes `daemon-preflight` read commands
- bridge-hosted daemon shell `Proof Diagnostics` panel now includes `daemon-preflight`
- local launcher summary now includes implementation-gate preflight checks
- getting-started now points to `daemon-preflight` alongside operator-phase and proof diagnostics

## Boundary

- this release does not accept the implementation gate
- this release does not add process control, IPC exposure, or OS service activation
- this release does not ship tray or background supervision UX
- `daemon-preflight` remains schema-only and non-mutating

## Recommended reads

```bash
sayane app daemon-operator-phase-status --json
sayane app daemon-preflight --json
sayane app daemon-preflight --json --include-event-record
sayane app daemon-proof-diagnostics --operation-class bridge_health --json
```
