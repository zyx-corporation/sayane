# Sayane v1.0.24 — Native Operator Summary Rail

## Summary

Sayane `v1.0.24` extends the native macOS daemon screen by adding the same compact operator summary
rail already introduced in the bridge-hosted daemon shell.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
lets native operators see the current gate, next command, and next read surface before scanning the
longer workspace, gate, and drill-down sections.

## Highlights

- native macOS daemon screen now shows `Operator Summary Rail` directly under the `Start Here` focus section
- the rail summarizes the current unresolved gate, the next suggested command, and the next read surface in one compact row
- each rail card reuses the same open-section and copy-command actions as the deeper daemon panels
- operator handoff and getting-started docs now treat native and bridge operator orientation as the same read-first workflow

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not replace the existing native workspace, gate guide, drill-down, or decision-assist sections
- this release does not change the current `cli_first_local_bridge` supported operator path
