# Sayane v1.0.30 — Native LaunchAgent Runbook Order

## Summary

Sayane `v1.0.30` extends the native macOS daemon screen by reordering the `LaunchAgent Runbook`
into the same health-first read order already established by the current-state section.

This still does **not** close the packaging and supervision phase. It stays read-first, but now
makes the Runbook start from preflight and verification before moving into logs and lower-level
preview detail.

## Highlights

- native macOS `LaunchAgent Runbook` now presents preflight, verification, and proof diagnostics before logs and preview detail
- log paths and tail commands now come immediately after the health-oriented checks they support
- environment assumptions, program arguments, preview metadata, and plist preview now follow after the main health checks
- getting-started, handoff, and native README guidance now describe the Runbook as a continuation of the current-state read path

## Boundary

- this release does not add mutating service lifecycle UI
- this release does not change the current `cli_first_local_bridge` supported operator path
- this release does not change the underlying CLI commands; it only changes how the native Runbook orders them
