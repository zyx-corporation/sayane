# Sayane v1.0.10 Resident Daemon Cleanup Decision Preview

Release date: 2026-06-17

## Summary

Sayane v1.0.10 packages the resident daemon cleanup decision preview track.

This release adds read-only stale artifact diagnostics and a non-mutating cleanup decision model for planned resident daemon runtime artifacts.

The resident daemon preview sequence is now:

```text
identity preview -> lifecycle preview -> operation plan preview -> runtime layout preview -> stale artifact diagnostic preview -> cleanup decision preview
```

- #199 Resident daemon stale artifact diagnostic preview

## Added commands

Stale artifact diagnostic preview:

```bash
sayane app daemon-stale-artifacts --json
```

Cleanup decision preview:

```bash
sayane app daemon-cleanup-decisions --json
```

Both commands support a runtime root override:

```bash
sayane app daemon-stale-artifacts --runtime-root /path/to/runtime --json
sayane app daemon-cleanup-decisions --runtime-root /path/to/runtime --json
```

## Diagnostic classifications

The diagnostic preview reports planned resident daemon artifacts as:

```text
missing
present_review_required
type_mismatch_review_required
```

The observed artifact set includes the runtime root, planned PID file, planned lock file, planned socket file, and planned runtime subdirectories.

## Cleanup decision rules

The decision preview currently maps diagnostics to conservative recommendations:

```text
missing -> no_action
present_review_required -> manual_review_required
type_mismatch_review_required -> unsafe_to_delete
```

No current decision reports an artifact as safe for automated cleanup.

## Safety boundary

This release remains preview-only.

The new commands observe and classify planned daemon artifacts. They do not modify runtime artifacts, acquire daemon locks, create IPC endpoints, or control daemon processes.

The payloads explicitly keep mutation indicators false:

```text
creates_artifacts: false
repairs_artifacts: false
deletes_artifacts: false
mutates_filesystem: false
```

## RDE Delta-M Review

### Preserved

Resident daemon work remains staged and auditable.

Preview commands remain non-mutating.

Manual review remains the default for ambiguity.

### Supplemented

The system now separates:

```text
observation -> decision -> mutation
```

Only observation and decision exist in v1.0.10.

Mutation remains out of scope.

### Deviation Risks

Do not treat preview recommendations as proof of process liveness.

Do not add mutation behavior to preview commands.

Do not reuse preview command names for a future mutating cleanup command.
