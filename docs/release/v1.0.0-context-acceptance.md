# Sayane v1.0.0 — Context Acceptance

## Summary

Sayane v1.0.0 is the first stable architecture release for local-first LLM context acceptance.

This release completes the Phase 3–19 implementation and documentation chain.

## What this release provides

- Candidate-based context import.
- Semantic review for overlap, unstable placement, and boundary-sensitive candidates.
- Human review workflow: approve, reject, modify, defer.
- Append-only audit trail for review decisions.
- Bundle provenance metadata and SHA-256 verification.
- Transfer regression dashboard.
- Built-in policy profiles: strict, standard, legacy-compatible, development.
- Custom policy files.
- Provenance-aware candidate review surfaces.
- Review decision diff viewer.
- Audit export in Markdown, JSON, and JSONL.
- Ed25519 signing and tamper detection.
- Signed export package workflow.
- Public narrative, architecture summary, diagrams, and CLI reference.

## What this release does not claim

- No automatic truth judgement.
- No automatic candidate acceptance.
- No automatic candidate rejection.
- No legal non-repudiation claim.
- No remote trust model.
- No organization key distribution.
- No encrypted export package.
- No cross-instance audit import.

## Test status

- 107 T-RDE tests.
- 462 full-suite tests passed.
- 0 Phase regressions.

## RDE note

This release transforms Sayane from a context portability prototype into a context acceptance and verifiable handoff system. It preserves the human review boundary and treats signatures as integrity evidence, not truth claims.
