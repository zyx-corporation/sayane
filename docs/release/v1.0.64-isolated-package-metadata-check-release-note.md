# v1.0.64 Isolated Package Metadata Check Release Note

## Summary

- moved local package metadata verification onto a short-lived isolated virtualenv
- replaced hanging local `twine check` runs with a direct `pkginfo` plus
  pure-Python Markdown rendering verification path
- kept normal build output unchanged: `python -m build` still produces the same
  `dist/` artifacts before metadata verification runs

## Why

- current release work should not fail just because the maintainer's long-lived
  development virtualenv contains a damaged native wheel or a hanging metadata
  checker
- package verification needs a reproducible path that is owned by the repo and
  can be re-created from scratch during release prep
