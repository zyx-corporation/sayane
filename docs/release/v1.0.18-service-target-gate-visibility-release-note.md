# Sayane v1.0.18 — Service Target Gate Visibility

## Summary

Sayane `v1.0.18` narrows the next operator-phase gap by making service-target policy gates more
visible in the bridge-hosted daemon shell and launcher guidance.

This still does **not** close the packaging and supervision phase. It only makes the blocking
service-target gate conditions easier to inspect from the same operator read stack.

## Highlights

- bridge-hosted daemon shell now shows service-target `policy_gates`
- bridge-hosted daemon shell now shows each target's `packaging_gate_status`
- launcher and getting-started now call out `daemon-service-targets-status --json` directly

## Boundary

- this release does not finalize the supported packaging model
- this release does not clear the hybrid packaging gate
- this release does not add service install / enable / disable UI

## Recommended reads

```bash
sayane app daemon-service-targets-status --json
sayane app daemon-operator-phase-status --json
sayane app daemon-preflight --json
sayane app daemon-proof-diagnostics --operation-class bridge_health --json
```
