# v1.0.40 Native Daemon Handoff Copy Actions Release Note

## Summary

The native macOS daemon screen now copies the current operator summary, phase-closure gates,
suggested actions, and read-surface references directly from the app.

## What changed

- added `運用サマリーレールをコピー` on the native daemon operator summary rail
- added `Phaseゲートをコピー` on the phase-closure gate guide
- added `推奨アクションをコピー` on the decision-assist block
- added `参照先をコピー` on the evidence drill-down block
- built the copied text from the current daemon screen-state payload so handoff text stays aligned
  with the same read-first operator context shown in the app
- kept the change inside the native macOS app surface only; no extension work was touched

## Operator impact

- daemon-side handoff no longer requires manual selection across multiple cards
- the current gate, next command, and next read surfaces can be moved into chat or notes with one action
- the copied text stays inside the existing local-first, read-first operator boundary

- `swift build --package-path macos/SayaneApp`
- focused `swift test --package-path macos/SayaneApp --filter ...` was attempted, but this environment
  still intermittently hangs in `swiftpm-xctest-helper` after build completion
