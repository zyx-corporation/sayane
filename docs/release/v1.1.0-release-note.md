# Sayane v1.1.0 — Scoped Context and Safe Reuse

## Summary

Sayane v1.1 extends the v1.0 Context Acceptance architecture with scoped context safeguards.

It addresses **Trojan context** risk: locally valid context that loses its scope or conditions and is later treated as global profile context.

## Highlights

- `scoped_accept` review decision with accepted scope, conditions, negative constraints, promotion policy, and reuse policy
- MCP scoped context output preserving scope / conditions / negative_constraints across all modes (compact / full / strict)
- Sidepanel scoped context review with Japanese UI labels
- Package import preview without modifying profiles
- Transfer regression HTML dashboard with scoped context, MCP boundary, and package preview sections
- PRFAQ and T-RDE public narrative checklist

## Architecture

```
v1.0: Context Acceptance
v1.1: v1.0 + scoped context defense layers
      F-1.5: 部分文脈を scoped_accept として封じ込める
      F-2:   MCP出力時に scope/conditions/negative_constraints を落とさない
      F-3:   候補確認時に scope/conditions/promotion risk を可視化
      F-4:   外部packageを import 前に preview
      F-5:   全リスクを HTML ダッシュボードで継続観測
```

## Boundary

- `scoped_accept` is not ordinary approval — it is conditional acceptance within a declared scope
- MCP output is derived context, not canonical profile
- Package Preview is not import — no profile is modified
- Dashboard is observation, not enforcement
- Verified package is not automatic acceptance
- T-RDE is a review discipline, not a truth oracle

## CLI

```bash
# Scoped context acceptance
sayane review scoped-accept <id> --scope project:target:subscope \
  --conditions "..." --negative "..." --reason "..."

# MCP compiled context
sayane context-compile --target cursor --mode full|compact|strict

# Package preview (no import)
sayane package preview pkg/

# HTML dashboard
sayane transfer-report --format html -o dashboard.html
```
